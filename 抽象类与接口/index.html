抽象类与接口

  • 抽象类
  • 接口
Java

如果一个类中包含抽象方法，那么这个类就是抽象类。在Java语言中，可以把类或者类中的某些方法声明为abstract来表示一个类是抽象类。接口是指一个方法的集合，接口中的所有方法都没有方法的具体实现，在Java中，接口是通过关键字interface来实现的。

抽象类和接口都是支持抽象类定义的两种机制。但是二者存在很大的区别。

只要包含一个抽象方法的类就必须被声明为抽象类，抽象类可以声明方法的存在而不去实现它，被声明为抽象的方法不能包含方法体。在实现时，必须包含相同或更低的访问级别（public、protected、private）。抽象类在使用过程中不能被实例化，但是可以创建一个对象使其指向具体子类的一个实例。抽象类的子类为父类中的所有抽象方法提供具体的实现，否则它们也是抽象类。

接口可以被看作抽象类的变体。接口中的所有方法都是抽象的，可以通过接口来间接地实现多重继承。接口中的成员变量都是static final类型。由于抽象类可以包含部分方法的实现，因此在一些场合下抽象类比接口存在更多的优势。

相同点：

  1. 都不能被实例化。
  2. 接口的实现类或抽象类的子类都只有实现了接口或抽象中的方法后才能被实例化。

不同点：

  1. 接口只有定义，其方法不能在接口中实现，只有实现接口的类才能实现接口中定义的方法，而抽象类可以有定义与实现，即其方法可以在抽象类中被实现。
  2. 接口需要实现，但抽象类只能被继承。一个类可以实现多个接口，但一个类只能继承一个抽象类，因此我们使用接口可以间接的实现多重继承的目的。
  3. 接口强调特定功能的实现，其设计理念是“has-a”关系；而抽象类强调所属类关系，其设计理念为“is-a”关系。
  4. 接口中定义的成员变量默认为public static final，只能够有静态的不能被修改的数据成员，而且必须给其赋初值，其所有成员方法都是public、abstract的，而且只能被这两个关键字修饰。而抽象类可以有自己的数据成员变量，也可以有非抽象的成员方法。
  5. 接口被用于实现比较常用的功能，便于日后维护或者添加删除方法；而抽象类更倾向于充当公共类的角色，不适用于日后重新对里面的代码进行修改。
