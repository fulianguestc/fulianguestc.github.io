再谈散列函数

  • hashCode()
Java

散列技术既是一种存储方法，也是一种查找方法。散列技术是在记录的存储位置和它的关键字之间建立一个确定的对应关系f，使得每个关键字key对应一个存储位置f(key)。查找时根据这个确定的对应关系找到给定值key的映射f(key)，若查找集合中存在这个记录，则必定在f(key)的位置上。

我们把这种对用关系f称为散列函数，又叫哈希函数。采用散列技术将记录存储在一块连续的存储空间中，这块连续存储空间称为散列表或哈希表。

散列函数的构造方法

一个好的散列函数有两个原则，计算简单和散列地址分布均匀。

直接定制法。取关键字的某个线性函数值为散列地址，即f(key)=a*key+b。

数字分析法。如果关键字是位数较多的数字，比如我们的11位手机号“130****1234”，前七位是有固定含义易重复，后四位才是真正的用户名分布均匀，可用作散列地址。即将后4位抽取出来。

平方取中法。假设关键字是1234，那么它的平方就是1522756，再抽取中间的3位数是227，用作散列地址。比较适合于不知道关键字的分布，而位数又不是很大的情况。

折叠法。折叠法是将关键字从左到右分割成位数相等的几部分，然后将这几部分叠加求和，并按散列表表长，取后几位作为散列地址。适合不需要知道关键字的分布，且关键字位数较多的情况。

除留余数法。此方法是最常用的构造散列函数方法。对于散列表为m的散列函数公式为：f(key)=key mod p。p一般取接近m的质数。

随机数法。当关键字长度不等时，可以直接用f(key) = random(key)。

处理散列冲突的方法。

当两个关键字key1!=key2时，但是确有f(key1)=f(key2)，这个现象即为散列冲突。

开放定址法。一旦发生了冲突，就去寻找下一个空的散列地址，只要散列表足够大，空的散列表总能找到，并将记录存入。f(key) = (f(key)+d) MOD m。(d=1,2,3,……,m-1)。

再散列函数法。f(key)=RH(key)。每当发生散列地址冲突时，就换一个散列函数计算，总有一个会把冲突解决掉。

链地址法。将所有关键字为同义词的记录存储在一个单链表中，散列表中只存储所有同义词的头指针。

公共溢出区法。当有关键字与之前的关键字位置有冲突，那么就将它们存储到溢出表中。在查找时，对给定值通过散列函数计算出散列地址后，先与基本表的相应位置进行对比，如果相等则查找成功，如果不等再去溢出表进行顺序查找。
