Linux命令

关于档案与目录的一些基本命令

档案与目录管理

cd（变换目录）

. 代表此层目录
.. 代表上一层目录
- 代表前一个工作目录
~ 代表【目前用户身份】所在的家目录

pwd（显示目前所在的目录）

-P 显示确切的目论

mkdir（建立新目录）

-p 建立多层目录
-m 建立带有权限的目录

rmdir（删除【空】的目录）

-p 连同上层空的目录一起删除

ls（档案与目录的检视）

-a 全部的档案，连同影藏档
-d 仅列出目录本身
-l 档案的属性与权限

cp（复制档案或目录）

-a 复制所有属性（一模一样）
-i 若目标文件已经存在时，在覆盖时会先询问动作的进行
-p 连通档案的属性一起复制过去，而非使用默认属性
-r 递归持续复制，用于目录的复制行为

rm（移除档案或目录）

-f 忽略不存在的文档，不会出现警告讯息
-i 在删除前会询问使用者是否动作
-r 很危险，会把该目录下所有内容都删除

mv(移动档案与目录，或更名)

-f 如果目标档案已经存在，不会询问而直接覆盖
-i 若目标档案存在，就会询问是否覆盖
-u 若目标档案存在，且source比较新，才会更新

cat（档案内容查阅） tac（反向列示）

-b 列出行号，空白行不标行号
-n 打印出行号，连同空白行也会有行号，与-b不同

档案权限

chgrp（改变档案所属群组）

要改变的组名必须要在/etc/group档案内存在才行，否则就会显示错误

-R 进行递归的持续变更

chown（改变档案拥有者）

-R 进行递归的持续变更

chmod（改变权限）

xyz 数字类型的权限属性，为rwx属性值的相加
-R 进行递归(recursive)的持续变更

Java中this的用法

最初接触Java时，就一直无法理解this的用法，觉得很多地方都有它的身影。下面就简单介绍一下this的集中基本用法。

主要参考书籍《Java核心技术.卷1》。

1. 隐式参数与显示参数

方法用于操作对象以及存取它们的实力域。例如，方法：

public void raiseSalary(double byPercent)
{
    double raise = salary * byPercent / 100;
    salary += raise;
}

raiseSalary方法有两个参数。第一个参数被称为隐式参数，是出现在方法名前的Employee类(本方法所在类)对象。第二个参数位于方法名后面括号中的数值，这是一个显式参数。

显示参数是明显地列在方法声明中的的显示参数，例如double byparent。隐式参数没有出现在方法声明中。

在每一个方法中，关键字this表示隐式参数。如果需要的话，可以用下列方式编写raiseSalary方法：

public void raiseSalary(double byPercent)
{
    double raise = this.salary*byPercent/100;
    this.salary += raise;
}

这样的好处是可以将实力域与局部变量明显地区分开来

2. 参数名

在编写很小的构造器时，常常在参数名上出现错误。通常，参数用单个字符命名：

public Employee(String n, double s)
{
    name = n;
    salary = s;
}

还一种常用的技巧,它基于这样的事实:参数变量用同样的名字将实例域屏蔽起来。例如,如果将参数命名为salary,salary将引用这个参数,而不是实例域。但是,可以采用this.salary的 形式访问实例域。回想一下,this指示隐式参数,也就是被构造的对象。下面是一个示例：

public Employee(String name, double salary)
{
    this.name = name;
    this.salary = salary;
}
3. 调用另一个构造器

关键字this引用方法的隐式参数。然而,这个关键字还有另外一个含义。

如果构造器的第一个语句形如this(…),这个构造器将调用同一个类的另一个构造器。下面是一个典型的例子:

public Employee(double s)
{
    // calls Employee(String, double)
    this("Employee #" + nextId, s);
    nextId;
}

当调用new Employee(60000)时,Employee(double)构造器将调用Employee(String, double)构造器。

采用这样的方法使用this关键字非常有用，这样对公共的构造器代码部分之编写一次即可。

4. this与super的异同

有些人认为super与this引用是类似的概念,实际上,这样比较并不太恰当。这是因 为super不是一个对象的引用,不能将super赋给另一个对象变量,它只是一个指示编译器 调用超类方法的特有关键字。

关键字this有两个用途:一是引用隐式参数,二是调用该类其他的构造 器。同样,super关键字也有两个用途:一是调用超类的方法,二是调用超类的构造器。 在调用构造器的时候,这两个关键字的使用方式很相似。调用构造器的语句只能作为另 一个构造器的第一条语句出现。构造参数既可以传递给本类(this)的其他构造器,也可 以传递给超类(super)的构造器。

关于adb-shell的若干问题

最近一直在进行安卓开发，在windows下启动adb shell只要配置环境变量就好了，并且网上教程很多，那么在MAC下如何配置呢？

大体思路是一致的，只是要用命令行来操。

首先启动终端，然后输入：

$ touch .bash_profile
$ open -e .bash_profile

这时会打开.bash_profile这个文档,

export PATH=${PATH}:/Users/***/sdk/platform-tools

在后面输入platform-tools所在的目录路径，比如笔者的路径就是/Users/***/sdk/platform-tools，所以写成上面的形式。然后直接关闭文档自动保存。

这时候在终端输入adb，就不会提示错误信息了，然后启动安卓模拟器，再次输入adb shell即可。
