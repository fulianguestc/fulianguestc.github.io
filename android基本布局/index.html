Android基本布局

  • UI布局
Android

LinearLayout

LinearLyout即线性布局，是我们最为常用的布局方式。这个布局会将它所包含的空间在线性方向上依次排列。那么下面即介绍一下这个方向属性：

android:orientation = "vertical"  //垂直方向
android:orientation = "horizontal" //水平方向（默认方向）

需要注意的是如果线性布局的排列方向是horizontal，内部的控件宽度就不能指定为match_parent，这样控件就会将水平方向占满，相应的如果排列方向是vertical，内部的控件就不能讲高度指定为match_parent。

下面介绍几个关键属性：

  • android:layout_gravity。用于指定控件在布局中的对齐方式。
  • android:gravity。用于指定文字在控件内部的对齐方式。
  • android:layout_weight。允许我们使用比例的方式来指定控件大小，在手机屏幕的适配性方面可以起到非常重要的作用。

RelativeLayout

RelativeLayout称作相对布局，是一种非常常用的布局。相对布局相对线性布局来说更加随意一些，它可以通过相对定位的方式让控件出现在布局的任何位置，所以在相对布局中的属性很多，不过也是相当有规律且易记得。

android:layout_alignParentLeft = "true"
android:layout_alignParentTop = "true"  //控件相对于父布局进行定位

android:layout_above = "@id/button"
android:layout_toLeftOf = "@id/button"  //控件相对于另一个控件的定位

android:layout_alignLeft = "@id/button" //一个控件的左边缘和另一个控件的左边缘对齐

android:layout_centerInParent = "true" //将控件置于父控件的中心位置
android:layout_centerHorizontal = "true" //将控件置于水平方向的中心位置

FrameLayout

FrameLayout布局没有任何的定位方式，所有的控件都会摆放在布局的左上角，主要是应用于Fragment的设计上。

TableLayout

TableLayout允许我们使用表格的方式来排列控件。在TableLayout中每加入一个TableRow就表示在表格中添加了一行，然后在TableRow中每加入一个控件就表示在该行中加入了一列，而且TableRow中的控件是不能指定宽度的。

android:layout_span = "2" //让控件占据两列的空间
android:stretchColumns = "i" //如果表格不能完全占满屏幕宽度，就将第i+1列进行拉伸。
