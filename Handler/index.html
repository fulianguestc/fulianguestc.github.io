异步消息处理机制

  • Handler
  • 异步消息
Android

上篇文章我们提到Android是不允许在子线程进行UI更新的，我们需要在子线程中之前执行一些耗时任务，然后根据任务的执行结果来更新相应的UI控件，为此Android提供了一套异步消息处理机制，完美的解决在子线程中进行UI操作的问题。我们先看一下具体用法。

Handler handler = new Handler(){
    public void handleMessage(Message msg){
        switch(msg.what){
            case ***:
            break;
        }
    }
};

new Thread(new Runnable(){
    void run(){
        Message msg = new Message();
        msg.what = ***;
        handler.sendMessage(msg);
    }
}).start();

简要地说，在子线程中通过sendMessage()方法将msg发出，在主线程中通过handleMessage()进行处理，进行UI操作。

Android的异步消息处理主要由四个部分组成，Message、Handler、MessageQueue和Looper。下面对这四部分进行一个简要的介绍。

  • Message。Message是在线程之间传递的消息，它可以在内部携带少量的信息，用于在不同的线程之间交换数据。
  • Handler。Handler就是处理者的意思，主要是用于发送和处理消息的。发送消息是用sendMessage()方法，而发出的消息经过一系列地辗转处理后，最终会传递到handleMessage()方法中。
  • MessageQueue。MessageQueue是消息队列的意思，主要用于存放所有通过Handler发送的消息，，这部分会一直存在于消息队列中，等等被处理，每个线程中只有一个MessageQueue对象。
  • Looper。Looper是每个线程中的MessageQueue的管家，调用Looper的loop()方法后，就会进入到一个无限循环当中，然后每当发现MessageQueue中存在一条消息，就会将它取出，并传递到Handler的handleMessage()方法中。每个线程中只有一个Looper对象。
