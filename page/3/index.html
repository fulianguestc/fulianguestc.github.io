hashCode的作用

有一种数据结构可以快速地查找所需要的对象，这就是散列表（hash table）。散列表为每个对象计算一个整数，称为散列码（hash code）。散列码是由对象的实例域产生的一个整数。hashcode方法的主要作用是为了配合基于散列的集合一起正常运行，这样的散列集合包括HashSet、HashMap以及HashTable。

当向集合中插入对象时，如何判别在集合中是否已经存在该对象了？（集合中不允许重复的元素存在）。如果采用equals方法逐一去比较，效率很低，这时我们采用hashcode方法，当集合要添加新的对象时，先调用这个对象的hashcode方法，得到对应的hashcode值，实际上在HashMap的具体实现中会用一个table保存已经存进去的对象的hashcode值，如果table中没有该hashcode值，它就可以直接存进去，不用进行任何比较了；如果存在该hashcode值，就调用它的equals方法与新元素进行比较，相同的话就不存了，不相同就散列其它的地址，所以这里存在一个冲突解决的问题，这样一来实际调用equals方法的次数就大大降低了。

那么我们能根据hashcode值判断两个对象是否相等吗？答案是不可以，因为不同的对象可能会生成相同的hashcode值。虽然不能根据hashcode值判断两个对象是否相等，但是如果两个对象的hashcode值不等，则必定是两个不同的对象。如果要判断两个对象是否真正相等，必须通过equals方法。所以要想在自定义类中比较两个对象是否相等，需要重新覆盖hashcode和equals方法。equals只能保证两个对象是在逻辑上是相等的，但是在hashmap进行get操作时，会因为得到的hashcode值不同而结果不同。

ArrayList、LinkedList、Vector的区别

这三者均为可伸缩数组，既可以动态改变长度的数组。

ArrayList和Vector都是基于存储元素的Object[] array来实现的，它们会在内存中开辟一块连续的空间来存储，由于数据存储是连续的，所以可以通过下标来访问元素，索引数据的速度比较快，但是在插入元素时执行得比较慢。ArrayList和Vector都有一个初始化容量的大小，当里面存储的元素超过这个大小时就需要动态的扩充存储空间。为了提高程序的效率，每次扩充容量不是扩充一个存储单元。Vector默认扩充为原来的2倍，而ArrayList默认扩充为原来的1.5倍。

ArrayList和Vector最大区别就是synchronization的使用，没有一个ArrayList的方法是同步的，而Vector得绝大多数方法都是直接或间接同步的，所以Vector是线程安全的，ArrayList不是线程安全的，所以Vector的性能略差一些。

LinkedList是采用双向列表来实现的，对数据的索引需要从头部开始遍历，所以随机访问效率比较低，但是插入元素时不需要对数据进行移动，所以插入效率较高，但它是非线性安全的容器。

  • LinkedList
  • Vector

String、StringBuffer与StringBuilder的区别

String是不可变类，String对象一旦被创建，其值将不能被改变，而StringBuffer是可变类，当对象被创建后仍然可以对其值进行修改。由于String是不可变类，因此适合在需要被共享的场合中使用，而当一个字符串需要被修改时，最好使用StringBuffer来实现。如果用一个String来保存一个经常被修改的字符串时，在字符串被修改时会生成很多无用的对象，由于这些无用的对象会被垃圾回收器回收，因此会影响程序的性能。

String与StringBuffer在初始化时时不同的。当实例化String时可以利用构造函数（String s=new String(“Hello”)）或者（String s = “Hello”）来初始化，而StringBuffer只能用构造函数来初始化。

String字符串修改实现的原理如下：当用String类型来对字符串进行修改时，其实现方法是首先创建一个StringBuffer，其次调用StringBuffer的append()方法，最后调用StringBuffer的toString()方法把结果返回。

String s =  "Hello";
s += "World";

以上代码等价于下述代码：

StringBuffer sb = new StringBuffer(s);
s.append("World");
s = sb.toString();

所以可以看出String的改变导致程序执行效率较低。

StringBuilder也是可以被修改的字符串，它与StringBuffer类似。但是StringBuider不是线程安全的，如果只是在单线程中使用字符串缓冲区，那么StringBuilder的效率更高。

  • StringBuffer
  • StringBuilder

Object中公有方法

参考书籍：《Java核心技术》、《Java编程思想》

Obeject是所有类的超类，java是一门单根继承的程序语言。所有类都是由Object类扩展而来的。

equals方法

用于检测一个对象是否等于另一个对象。在Object类中，这个方法用于判断两个对象是否具有相同的引用。如果两个对象具有相同的引用，它们一定是相等的。

下面看一个例子：

class Test{
    int index;
}

public class TestEqual{
    public static void main(String[] args){
        Test test1 = new Test();
        Test test2 = new Test();
        System.out.print(test1.equals(test2));
    }
}

这个测试的结果是false。这是因为equals方法是比较引用的，所以除非在自己的新类中覆盖equals方法，否则不可能实现出相等的结果。

switch能否用String作为参数

case标签必须是整数或枚举常量，不能测试字符串。例如，下面这段代码就存在错误。

String input = ...;
switch(input){ //ERROR
    case "A": //ERROR
        ...
        break;
    ...
}

但是从Java SE 7开始，case标签还可以是字符串字面量。也就是像上面的写法在Java SE 7中是没有问题的。
