Object中公有方法

参考书籍：《Java核心技术》、《Java编程思想》

Obeject是所有类的超类，java是一门单根继承的程序语言。所有类都是由Object类扩展而来的。

equals方法

用于检测一个对象是否等于另一个对象。在Object类中，这个方法用于判断两个对象是否具有相同的引用。如果两个对象具有相同的引用，它们一定是相等的。

下面看一个例子：

class Test{
    int index;
}

public class TestEqual{
    public static void main(String[] args){
        Test test1 = new Test();
        Test test2 = new Test();
        System.out.print(test1.equals(test2));
    }
}

这个测试的结果是false。这是因为equals方法是比较引用的，所以除非在自己的新类中覆盖equals方法，否则不可能实现出相等的结果。

switch能否用String作为参数

case标签必须是整数或枚举常量，不能测试字符串。例如，下面这段代码就存在错误。

String input = ...;
switch(input){ //ERROR
    case "A": //ERROR
        ...
        break;
    ...
}

但是从Java SE 7开始，case标签还可以是字符串字面量。也就是像上面的写法在Java SE 7中是没有问题的。

Java中的四种引用

  • 文章出自Understanding；
  • 译文出自技术小黑屋，感谢分享。
强引用(Strong Reference)

强引用就是我们经常使用的引用，其写法如下:

StringBuffer buffer = new StringBuffer();

上面创建了一个StringBuffer对象，并将这个对象的（强）引用存到变量buffer中。是的，就是这个小儿科的操作（请原谅我这样的说法）。强引用最重要的就是它能够让引用变得强（Strong），这就决定了它和垃圾回收器的交互。具体来说，如果一个对象通过一串强引用链接可到达(Strongly reachable)，它是不会被回收的。如果你不想让你正在使用的对象被回收，这就正是你所需要的。

《拆书帮：这样读书就够了》（二）

书中案例反思

（1）专业的一个基础要求：注重事实，关注细节。

例如，给老板汇报，和人争论时。
拿出细节事实和证据，不直接说观点（说事实比说观点更有效，更能影响人）。
说事实的要点是：讲出细节（细节能说服人、打动人）。
例如：HR招聘，会问细节，STAR法则。
例如：销售：说细节，说数据，说成功故事（越细越好）。

（2）分类显示专业。

例如：和老板汇报。
跟领导说项目遇到的具体困难，有几种（先分类：可控，不可控，无法预测的），这次遇到的是……+困难的细节和原因。

（3）展现专业之外的周全和可靠

对于特别重要的事情，涉及到决策层面，仅仅展示客观事实就不够了。除了好员工的专业，还需要展现周全（思维全面）和可靠（可以放心托付重要事项）——可以提拔为候选人。
一份正式的用于决策的汇报=事实（有细节）+明确的观点+明确具体的建议（至少2个）+行动后的预测（对于实施效果的积极和消极两种预测）即如果选择A，有什么好处，可能出什么意外；如果选择B，有什么好处，可能出什么意外。

Java的基本数据类型以及它们的封装类

Java是一种强类型语言，所以需要为每一个变量声明一种类型。在Java中，一共有8种基本类型，其中有4种整型、2种浮点类型、1钟字符类型char和1种表示真值的boolean类型。

备注：Java有一个能够表示任意精度的算术包，通常称为“大数值”。但它并不是一种新的Java类型，而是一种Java对象。

整型

提供了4种整形，允许时负数，而且不存在无符号类型。int，4字节，32位；short，2字节，16位；long，8字节，64位；byte，1字节，4位。

byte和short类型主要用于特定场合，例如，底层的文件处理或者需要控制占用存储空间量的大数组。整形范围是固定的，不会有软件移植的问题。
