多线程简介

Thread

当我们需要执行一些耗时操作，比如发起一条网络请求时，考虑到网速等其他原因，服务器未必会立刻响应我们的要求，如果不将这类操作放在子线程里去运行，就会导致主线程被阻塞，从而影响用户对软件的正常使用，下面介绍线程的两种基本用法。

class MyThread extends Thread{
    @Override
    public void run(){
        //处理具体逻辑
    }
}
new MyThread().start();

new Thread(new Runnable(){
    @Override
    public void run(){
        //处理具体逻辑
    }
}).start()；

Android的UI是线程不安全的，也就是说，如果想要更新应用程序里的UI元素，则必须在主线程进行，否则就会出现异常。但是有些时候我们必须在子线程中去执行一些耗时的任务，然后根据任务的执行结果来更新相应的UI控件，为此，Android提供了一套异步消息处理机制，我们在后面会专门分析这套模型。

泛型小问题

泛型简介

具有一个或多个类型参数的类或者接口就是泛型类或者接口。泛型类和接口统称为泛型。每个泛型都定义一个原生态类型，即不带任何实际类型参数的泛型名称。例如List相对应的原生态类型就是List。

泛型可以告诉编译器每个集合接受哪些对象类型，编译器自动地为为插入的内容进行转化，并在编译时告知是否插入了类型错误的对象，这样可以使程序更安全更清楚。

如果不提供类型参数，使用集合类型和其他泛型仍然是合法的，但是不应该这么做，会破坏程序的安全性。保留原生态类型是为从前没有使用泛型的代码提供兼容性。

泛型有子类型化得规则，List<\String>是原生态类型List的一个子类，而不是参数化List<\Object>的子类，所以二者不能相互转化。

Android布局优化

布局重用include

我们想象这样一种情况，一个APP的应用内每个界面都有一个标题栏上面有一个back按钮。那么我们是不是需要在每个界面都要重复写一遍这样的标题栏代码呢，这样就会导致代码的大量重复，这个时候我们就可以采用include方式来解决这个问题。

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"  
    android:orientation="vertical"   
    android:layout_width=”match_parent”  
    android:layout_height=”match_parent”>    
<include layout="@layout/title"/> 
</LinearLayout>

title.xml是我们的标题栏代码，只需在每个活动界面用include引入title.xml，不需要重复编码。

减少视图层级merge

merge标签用于减少View树的层次来优化Android的布局，在UI结构优化中起着非常重要的作用，它可以删除多余的层级。merge多用于替换FrameLayout或者当一个布局包含另一个时，merge标签消除视图层次结构中多余的视图组。

需要时使用ViewStub

ViewStub标签最大的优点是当你需要时才会加载，使用他并不会影响UI初始化时的性能。各种不常用的布局像进度条、显示错误消息等可以使用该标签，以减少内存使用量，加快渲染速度。ViewStub是一个不可见的，大小为0的View。

程序在运行期间，某个布局在inflate()后，就不会有变化，除非重新启动。所以当需要在运行时不止一次的显示和隐藏某个布局，ViewStub是做不到的，只能用View的可见性来控制。

XML和JSON简介

通常情况下，每个需要访问网络的应用程序都会有一个自己的服务器。我们可以向服务器提交数据，也可以从服务器上获取数据，所以一般我们会在网络上传输一些格式化后的数据，这种数据会有一定的结构规格和语义，当另一方收到数据消息之后就可以按照相同的结构规格进行解析，从而取出想要的那部分内容。在网络上传输数据时最常用的格式有两种，XML和JSON，接下来我们就学习一下这两种方式的解析方法。

XML

解析XML格式的数据有很多种方式，下面介绍比较常用的两种，Pull解析和SAX解析。

Pull解析方式

XmlPullParserFactory factory = XmlPullParserFactory.newInstance();
XmlPullParser xmlPullParser = factory.newPullparser();
xmlPullParser.setInput(new StringReader(xmlData));
int eventType = xmlPullParser.getEventType();//得到当前的解析事件
String id = "";String name = "";String version = "";
while(eventType != XmlpullParser.END_DOCUMENT){
    String nodeName = xmlPullParser.getName();
    switch(eventType){
        //开始解析某个节点
        case XmlPullParser.START_TAG:
            if("id".equals(nodeName)){
                id = xmlPullParser.nextText();
            }else if("name",equals(nodeName)){
                name = xmlPullParser.nextText();
            }else if("version",equals(nodeName)){
                version = xmlPullParser.nextText();
            break;
        //完成解析某个节点
        case XmlPullParser.END_TAG:
            break;
        default:
            break;
    }
    eventType = xmlPullParser.next();
}

Android基本布局

LinearLayout

LinearLyout即线性布局，是我们最为常用的布局方式。这个布局会将它所包含的空间在线性方向上依次排列。那么下面即介绍一下这个方向属性：

android:orientation = "vertical"  //垂直方向
android:orientation = "horizontal" //水平方向（默认方向）

需要注意的是如果线性布局的排列方向是horizontal，内部的控件宽度就不能指定为match_parent，这样控件就会将水平方向占满，相应的如果排列方向是vertical，内部的控件就不能讲高度指定为match_parent。

下面介绍几个关键属性：

  • android:layout_gravity。用于指定控件在布局中的对齐方式。
  • android:gravity。用于指定文字在控件内部的对齐方式。
  • android:layout_weight。允许我们使用比例的方式来指定控件大小，在手机屏幕的适配性方面可以起到非常重要的作用。

SQLite数据库操作

创建数据库

Android为了方便我们能够方便的管理数据库，专门提供了一个SQLiteOpenHelper帮助类，借助这个类进行数据库的创建与升级。

SQLiteOpenHelper类是一个抽象类，我们需要创建一个自己的帮助类来继承它，重写抽象方法onCreate()和onUpgrade()，实现创建和升级的逻辑。还有另外两个重要的方法getReadableDatabase()和getWritableDatabase()。这两个方法都可以创建或打开一个现有的数据库，并返回一个可对数据库读写的对象。不同的是当数据库不可写入时，前者返回的对象将以只读的方式打开数据库，后者将会出现异常。

SQLiteOpenHelper的构造方法有四个参数，第一个参数是Context，第二个参数是数据库名，第三个参数允许我们在查询数据的时候返回一个自定义Cursor，一般传入null，第四个参数表示当前数据库的版本号，用于对数据库进行升级操作。构建出SQLiteOpenHelper实例后，再调用getReadableDatabase()或getWritableDatabase()方法，会返回一个SQLiteDatabase对象，方便之后的操作，此时onCreate方法会得到执行。

Android数据存储形式

文件存储

文件存储是Android中最基本的一种数据存储方式，不对内容进行任何的格式化处理，所有数据都原封不动的保存到文件当中，适合存储一些简单的文本。

将数据存储到文件中

利用openFileOutput()方法创建FileOutputStream对象，用于将数据存储到指定的文件中。这个方法接收两个参数，第一个参数是文件名，第二个参数是文件的操作模式，主要有MODE_PRIVATE和MODE_APPEND两种模式。其中MODE_PRIVATE是当指定同样文件名时所写入内容将会覆盖原文件中的内容，而MODE_APPEND是向已有文件中追加内容。

读取数据与写入数据类似，主要使用openFileInput()方法，然后利用Java的各种流的知识进行读写操作。

SharedPreferences存储

SharedPreferences存储是使用键值对的方式来存储数据的，也就是当保存一条数据时，需要给这个数据提供一个对应的键值，这样在读取的时候就可以通过这个键将相应的数据读取出来。

ContentProvider介绍

内容提供器主要用于在不同的应用程序之间实现数据共享的功能，它提供了一套完整的机制，允许一个程序访问另一个程序中的数据，同时还能保证被访数据的安全性。目前，使用内容提供器是Android实现跨程序共享数据的标准方式。

内容提供器可以选择只对哪一部分数据进行共享，从而保证程序中的隐私数据不会有泄露的风险。

那么如何创建内容提供器为自己的程序数据提供外部访问接口呢？

内容提供器，可以通过新建一个类去继承ContentProvider的方式来创建一个自己的内容提供器。ContentProvider类中有六个抽象方法，我们需要将这六个方法全部重写。方法内的具体逻辑与数据库的实现方法很相似。

onCreate()

初始化内容提供器的时候调用。通常会在这里完成对数据库的创建和升级等操作，返回true表示内容提供器初始化成功，返回false表示失败。只有当存在ContentResolver尝试访问我们程序中的数据时，内容提供器才会被初始化。

query()

从内容提供器中查询数据。使用uri参数来确定查询哪张表，projection参数用于确定查询哪些列，selection和selectionArgs参数用于约束查询哪些行，sortOrder参数用于对结果进行排序，查询的结果存放在Cursor对象中返回。
