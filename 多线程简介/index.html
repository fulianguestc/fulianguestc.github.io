Android

Thread

当我们需要执行一些耗时操作，比如发起一条网络请求时，考虑到网速等其他原因，服务器未必会立刻响应我们的要求，如果不将这类操作放在子线程里去运行，就会导致主线程被阻塞，从而影响用户对软件的正常使用，下面介绍线程的两种基本用法。

class MyThread extends Thread{
    @Override
    public void run(){
        //处理具体逻辑
    }
}
new MyThread().start();

new Thread(new Runnable(){
    @Override
    public void run(){
        //处理具体逻辑
    }
}).start()；

Android的UI是线程不安全的，也就是说，如果想要更新应用程序里的UI元素，则必须在主线程进行，否则就会出现异常。但是有些时候我们必须在子线程中去执行一些耗时的任务，然后根据任务的执行结果来更新相应的UI控件，为此，Android提供了一套异步消息处理机制，我们在后面会专门分析这套模型。

AsyncTask

除了刚才提过的异步消息处理机制，还有没有更方便的工具呢？答案是当然有，AsyncTask就是其中之一，借助AsyncTask，即使你对异步消息处理机制完全不了解，也可以简单的从子线程切换到主线程。

AsyncTask是一个抽象类，所以如果我们想使用它，就必须创建一个子类去继承它，在继承时我们要指定三个泛型参数。

  • Params。在执行AsyncTask时需要传入的参数，可以在后台服务中使用。
  • Progress。后台任务执行时，如果需要在界面上显示当前的进度，则使用这里指定的泛型作为进度单位。
  • Result。当任务执行完毕后，如果需要对结果进行返回，则使用这里指定的泛型作为返回值类型。

我们还要去重写AsyncTask中的几个方法才能完成对任务的定制。

  • onPreExecute()。这个方法在后台任务开始之前调用，用于进行一些界面上的初始化操作，比如显示一个进度对话框等。
  • doInBackground(Params…)。这个方法的所有代码都会在子线程中运行，在这里处理所有的耗时任务。在这个方法中不能进行UI操作，如果需要更新UI操作，可以调用publishProgress(Progress…)方法来完成。
  • onProgressUpdate(Progress…)。在后台任务中调用了publishProgress方法后，这个方法就会很快被调用，方法中携带的参数就是从后台传递过来的，在这个方法中可以对UI进行操作，利用参数就可对界面更新。
  • onPostExecute(Result)。当后台任务执行完毕后通过return语句返回时，这个方法就会被调用，可以进行一些UI操作。

如果想要启动这个任务，只需运行一下代码：

new MyTask().execute();

IntentService

服务中的代码默认都是运行在主程序中的，如果我们在服务里处理一些耗时的逻辑，就会出现问题。所以我们在服务的每个方法中开启子线程，这种服务一旦启动起来就会一直处于运行状态，必须调用stopService或者stopSelf方法才能让服务停止下来。

这种方法很简单，但是总有一些程序员忘记开启线程，或者忘记调用stopself，就会出现问题。为此可以简单的创建一个异步的、会自动停止的服务，Android专门提供了一个IntentService类，这个类可以很好的解决前面的问题。

新建一个类继承IntentService，然后提供一个无参的构造函数，并且必须在其内部调用父类的有参构造函数，然后要在子类中实现onHandleIntent()抽象方法，在这个方法中可以去处理一些具体的逻辑，而且不用担心ANR问题。
